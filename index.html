Code Background

This is research code that delves into different potential uses for intel's Iris Graphics Pro. It focuses on the Pixel Ordering feature, as this sort of feature is now being implemented in other hardware (for instance, nVidia's pixel interlocking). What Pixel Ordering provides for us, is a defined and ordered access of shared memory across the pixel/fragment shader. Since we now have this guarantee of access, we can now do more informed calculations, especially when dealing with depth. This project aims to present techniques that utilize this feature to create higher quality, or more efficient renders of their traditional counterparts. To run this code, the iris extensions must be present in your hardware. The code will compile, but it will stop you after it pings the graphics driver if the program cannot find what it needs.

Progress So Far

So far, I have two render passes to a flat colorscale working. Since I now know that the first pass is gathering information correctly, I can now move on to practical graphical applications.
